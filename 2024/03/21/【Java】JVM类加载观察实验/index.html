实验部分

加载的实验

实现一个自定义的类加载器MyClassLoader。（注意这个类如果不写在实验类ClassLoaderExperiment里，那么执行javac的时候就要在项目的根目录下执行，不然会显示找不到类）

使用javac命令编译 ClassA.java、javac org/example/classLoader/ClassLoaderExperiment.java。

接着启动JVM运行Main方法

java org.example.classLoader.ClassLoaderExperiment。注意执行java函数的地址应该在项目根地址下。

实验观察

1. javac编译代码

首先实验是使用javac进行代码的编译，生成字节码。

2. 执行java xxx

接下来执行java xxx就是启动JVM运行main方法。这个过程中涉及到类的加载、链接和初始化。

  1. 启动JVM:

     - 操作系统创建一个新的JVM进程。JVM读取命令行参数和环境变量。

  2. 加载和链接启动类:

     - JVM的引导类加载器(Bootstrap Class Loader)加载核心Java类库,如 java.lang.Object 和 java.lang.String。

 - 引导类加载器使用C++实现,是JVM的一部分。

  3. 加载和链接用户类:

     - JVM的应用类加载器(Application Class Loader)加载用户类路径(classpath)上的类。
 这包括自己编写的类和第三方库。

  4. 执行main方法:

     - JVM查找 ClassLoaderExperiment 类的 main 方法,并开始执行它。main 方法是Java程序的入口点。

展开JVM类加载的步骤：

  1. 加载

     - 类加载器查找类的字节码(通常在 .class 文件中),并从中创建一个 Class 对象。

当 MyClassLoader 的 loadClass 方法被调用时,它触发了自定义类的加载过程。

  1. 链接

     - 验证:检查字节码的正确性和安全性。为了确保不会加载可能会破坏JVM内部结构的类。
 - 准备:JVM为类的静态字段分配内存,并用默认值初始化它们。
 - 解析:将符号引用转换为直接引用。其中JVM将类、接口、字段和方法的符号引用转换为直接引用。符号引用来自类或接口的常量池，而直接引用指向方法区中的地址位置。
 - 初始化：执行类的静态初始化器和静态字段的初始化表达式。
